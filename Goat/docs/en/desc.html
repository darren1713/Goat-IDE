"The Go @ IDE" is my personal contribution to the free software in the hope that it will be useful to those, passionate as me about microcontrollers and embedded systems, do not want (or can not) spend money on commercial software.
"The Go @ IDE" wants to be the junction point between hardware and software, combining features of software specific hardware tools,
(terminal emulator, dfu-programmer,...) with a powerful and highly customizable IDE
Goat currently includes the support for:
MicroPython
eLua
Arduino (the installation of the "Arduino IDE 1.0" is required)

My goal is make it grow in the wake of the suggestions of those who will use.
Use it and please send tips, comments and suggestions ...

Some features:
  • Its editor (based on the wonderful library "scintilla" http://www.scintilla.org/), is lightweight and powerful.

  • Has additional features such as "syntax highlighting", "code folding" and "code completion" based on the API of the language we are using.

  • Includes the capability to connect with the real hardware (through the serial or the USB port) obtaining the prompt of the target(*) inside it.

  • It's able to run the .py file on the Hardware (using the raw REPL interface).

  • It's able to send the .lua/.lc file on the Hardware (using the eLua xmodem protocol).

  • Supports some terminal commands (standard ansi/vt100), so it's able to play games like "hanoi" or "hungman" (using the TERM capability).

  • Supports the FLASH (on the target(*)) programming through the DFU/Arduino boot loader.

  • Supports easily the libusb through Lua scripts (experimental).

  • It's easily extensible through Lua scripts.

  • Almost everything that SciTE is able to do.

target(*) is a small hardware with a microcontroller, managed by an RTOS
and able to run a monolithic but very fast application.
Some examples are:

MicroPython (the pyboard)
The Mizar32 (low-cost Stackable Multi-Board Computer)
STM32F4Dicscover board
Arduino
